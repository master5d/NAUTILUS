# Vision & Principles

The SOVRN project is built on the philosophy of **Sovereign Solo Vibe Coding**. This approach prioritizes individual autonomy, local control, and high-velocity development.

## Core Principles

### 1. Zero Telemetry & Local First
We reject the "Software as a Service" model for our brains. All core data processing — indexing, vector extraction, and graph storage — happens locally or within your sovereign cloud (WSL/Local Storage). External LLMs are treated as stateless compute units, not repositories of truth.

### 2. Memory as a Data Mesh (Data-as-a-Product)
Data should not be a "lake" of raw files. It should be a structured "Mesh."
- **Immutable Facts**: Every record has an audit trail.
- **Contract-First**: Data is defined by schemas, not guesses.
- **Sovereign Control**: You own the index, the graph, and the keys.

### 3. Orchestrator-in-Chief (Hermes Role)
Hermes is the gatekeeper. It ensures that sub-agents (Coders, Researchers) interact with data only through strictly defined context windows. This prevents "context sprawl" and protects the integrity of the architecture.

### 4. Federated Ownership
Data belongs to domains (e.g., `tech/sovrn`, `knowledge/para`). Cross-domain connections are managed via federated links, creating a web of knowledge that reflects the real-world complexity of your work.

### 5. Vibe Coding Synergy
The system is designed to minimize friction. Tools should work together "by default," allowing the developer to focus on the "vibe" (the high-level intent) while the system handles the metadata and connectivity.
